# RedGPU V4.2.0-Alpha-rnd Bug Fixes Report

본 문서는 `V4.1.0-Alpha-rnd` 릴리즈 분기점 이후 `V4.2.0-Alpha-rnd` 개발 과정에서 해결된 주요 오류 및 버그 픽스(Bug Fix) 내역을 일목요연하게 정리한 리포트입니다.

---

### 1. ClusterLightManager 초기화 순서 크래시 해결

* **이슈 (AS-IS)**:
    - 엔진 렌더러가 최초 가동되며 바인드 그룹을 빌드하는 시점에, 빛 연산을 관리하는 `ClusterLightManager`가 완전히 활성화되지 않아 내부 패스(`PassClustersLight` 등)가
      `undefined` 상태로 접근되어 앱이 먹통이 되는 크래시가 유발되었습니다.
* **해결 (TO-BE)**:
    - 초기화 라이프사이클을 전면 재조정하여, 라이트 매니저가 가동에 필요한 종속성 패스들을 생성자 인수로 직접 전달받아 생성되도록 의존성 주입(DI) 구조로 전면 교정하여 초기화 순서 크래시를 원천
      차단했습니다.

### 2. 조작기(FreeController)의 카메라 회전각 동기화 오류 픽스

* **이슈 (AS-IS)**:
    - 자유 카메라 오프셋 조작기(`FreeController`)를 사용할 때 메시 컴포넌트의 고유 회전값(`rotationX`, `rotationY`)과 실제 카메라의 시선 회전각이 서로 동기화되지 않고 완전히
      따로 노는 버그가 있었습니다.
* **해결 (TO-BE)**:
    - 메시 조작 로직 변경 시 변화된 회전 정보를 카메라 회전 컴포넌트에 즉각 파이프라인으로 전송해 주는 회전각 양방향 동기화 처리를 추가하여 자연스러운 뷰 조작을 구현했습니다.

### 3. 카메라 줌(FOV) 및 클리핑 평면 변동 시 클러스터 라이트 영역 미갱신 오류 해결

* **이슈 (AS-IS)**:
    - 카메라의 줌 조작(fieldOfView)이나 근/원거리 잘림 평면(near/farClipping)이 변경되어 뷰 프러스텀 경계면이 급격히 틀어짐에도 불구하고, 오직 화면 해상도 픽셀 크기 변화에만 반응하여
      클러스터 AABB 그리드 버퍼가 이전 줌 값으로 묶여 고정되어 있었습니다. 이로 인해 줌을 당겼을 때 조명이 어긋나거나 깜빡이는 현상이 있었습니다.
* **해결 (TO-BE)**:
    - `ClusterLightManager` 내부에 `fov`, `near`, `far` 변경 감지 멤버 변수(#prevFOV, #prevNear, #prevFar) 및 더티 체크 조건식을 추가하여, 줌 배율
      변경 발생 시 뷰 공간 복셀 경계 정보가 실시간으로 재구축되도록 수정을 완료했습니다.

### 4. 카메라 이동/회전 시 실시간 컬링 판정 지연 및 영역 반영 오류 해결

* **이슈 (AS-IS)**:
    - 카메라가 조작기(OrbitController 등)에 의해 회전하거나 느리게 이동할 때, 프레임 연산 분산을 위한 인터리브드 컬링 조건식(`needCheckInterleavedCulling`)이 4프레임 중
      1번만 구동되도록 잘못 잠겨 있었습니다. 이로 인해 카메라 회전 시 화면 바깥에서 들어오는 오브젝트나 겹치는 조명 연산의 반응이 4프레임 딜레이되며 화면에서 껌뻑이고 밀렸고, 마우스 휠 줌(Zoom)을
      강하게 조작하여 고속 이동 임계치를 넘길 때만 강제 전수 검사가 유발되어 줌을 해야만 영역이 갱신되는 이상 동작이 있었습니다.
* **해결 (TO-BE)**:
    - `RenderViewStateData.ts` 의 조건식을 단순화하여, 카메라가 완벽하게 정지된 상태가 아닌 미세 움직임이라도 있는 동안에는 `forceCullingCheck`를 즉각 작동시켜 매 프레임
      정밀하게 100% 실시간 가시성 컬링을 수행하고, 카메라가 멈췄을 때만 컬링 연산을 통째로 스킵(0% 부하)하도록 성능과 정밀도를 모두 챙기도록 수정했습니다.
