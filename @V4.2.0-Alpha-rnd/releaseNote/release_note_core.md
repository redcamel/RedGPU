# RedGPU V4.2.0-Alpha-rnd Core Release Note (4대 기술 혁신)

이번 `V4.2.0-Alpha-rnd` 릴리즈는 웹 3D 그래픽스의 한계를 넘어서는 성능 최적화와 시각적 아름다움을 목표로 개발되었습니다. 이를 위해 핵심 뼈대가 되는 **4대 기술 혁신(통합 데이터 보관소
SSBO, 똑똑한 CPU 컬링, 부드러운 모션 블렌딩, 고품질 동적 그림자)**을 중심으로 엔진 코어가 대대적으로 개선되었습니다.

---

## 1. SSBO (Shader Storage Buffer Object) 구조 통합 및 리팩토링

**"물체 데이터를 매번 쪼개서 보내지 않고, 하나의 큰 보관소에 모아두고 고속으로 꺼내 씁니다."**

기존에는 화면에 물체를 그릴 때마다 해당 물체의 위치와 재질 정보를 매번 GPU에 일일이 알려주어야 해서 CPU와 GPU 사이의 통신 비용(바인드 그룹 교체 오버헤드)이 매우 컸습니다.

* **인덱스 기반 데이터 바인딩**:
    - 씬에 있는 모든 물체 정보를 컨텍스트 레벨의 거대한 단일 버퍼(`globalVertexSSBO`, `globalFragmentSSBO`)에 한꺼번에 담아둡니다.
    - 물체를 그릴 때는 버퍼를 통째로 고정해 두고, 물체마다 고유한 **'인덱스 번호표'**만 전달하여 셰이더 내에서 배열 형태로 고속 조회하게 만들었습니다. 이로 인해 그리는 속도가 비약적으로 향상되었습니다.

| 버퍼 명칭                            | 초기 슬롯 수 | 단일 슬롯 크기 (Bytes) | 주 용도 및 관리 대상                                            |
|:---------------------------------|:--------|:-----------------|:--------------------------------------------------------|
| **`globalVertexSSBO`**           | 2,000   | 304              | 메쉬 변환 행렬(`MatrixList`), Opacity, 피킹 ID 등 버텍스 연산 속성      |
| **`globalFragmentSSBO_PBR`**     | 1,024   | 912              | PBR 재질 속성 및 GLTF KHR 확장 스펙 파라미터 제어                      |
| **`globalFragmentSSBO_BuiltIn`** | 1,024   | 112              | 빌트인 기본 머티리얼(`ColorMaterial`, `BitmapMaterial` 등)의 속성 제어 |

* **현대적인 버퍼 확장 모델 (하이브리드 리사이징)**:
    - 이번 `V4.2.0-Alpha-rnd` 버전에 글로벌 스토리지 버퍼 매니저(`GlobalStorageBufferManager`)를 신설하며, 현대적인 웹 3D 렌더러에 적합한 하이브리드 리사이징 구조를
      도입했습니다.
    - **32MB 이하**일 때는 2배씩 빠르게 확장하여 할당 빈도를 낮추고, **32MB를 초과**하면 8MB 단위로만 점진 증가하도록 설계하여 모바일 및 브라우저 환경에서 발생할 수 있는 비정상 종료(
      OOM) 리스크를 최소화했습니다.
* **영리한 변경 감지 (더티 트래킹)**:
    - 메모리 내부에서 값이 실제로 변경된 범위만 콕 집어서 GPU로 전송함으로써 불필요하고 비효율적인 전송을 사전에 예방합니다.

## 2. CPU 컬링(Culling) 최적화 시스템

**"보이지 않거나 너무 작아서 안 보이는 물체는 계산에서 즉시 제외합니다."**

기존 버전에서는 씬에 있는 모든 물체를 대상으로 매 프레임 무조건적인 정밀 가시성 추적(6개 평면 충돌 검사)을 수행하여 성능 저하의 원인이 되었습니다. 이를 극복하고 필요한 물체만 영리하게 선별하여 연산 효율을
극대화하는 CPU 컬링 최적화 시스템을 새롭게 설계하여 이식했습니다.

| 컬링 필터 단계 및 기법                    | 기존 방식 (AS-IS)                    | 이번 버전에 새로 도입된 방식 (TO-BE)                                      | 연산 최적화 효과                    |
|:---------------------------------|:---------------------------------|:--------------------------------------------------------------|:-----------------------------|
| **① 후방 물체 즉시 제외 (Early-Out)**    | 모든 물체에 대해 6개 평면 충돌 정밀 내적 계산 수행   | 단순 1회 내적(Dot Product) 연산으로 카메라 뒤편에 있는 물체를 먼저 걸러냄              | 6-Plane 정밀 연산 비용을 원천 배제      |
| **② 연산량 나눠서 계산 (가변 인터리브드 컬링)**   | 매 프레임 모든 물체를 대상으로 무조건적인 전수 정밀 추적 | 물체들을 4개 그룹으로 분할하여 매 프레임 1/4씩만 계산하고 캐시 재사용                     | 카메라 속도에 비례해 연산량을 분산하여 부하 세이브 |
| **③ 원거리 아주 작은 물체 생략 (투영 면적 컬링)** | 물체 크기 및 거리와 상관없이 화면에 무조건 렌더링 시도  | 화면 높이 대비 크기 비율이 **1% 미만(`minScreenSpaceSize = 0.01`)**인 물체 생략 | 불필요한 렌더패스 제출 및 드로우콜 비용 차단    |

## 3. 모션 블렌딩 및 애니메이션 시스템 고도화

**"애니메이션 상태 머신(AnimStateMachine)을 구현하여 다중 모션을 유기적으로 연결하고 조작합니다."**

캐릭터가 가만히 서 있다가 걷거나 뛸 때, 혹은 동작이 전환될 때의 어색함을 없애고 다중 모션이 물 흐르듯 자연스럽게 섞이도록 개선했습니다.

* **애니메이션 상태 머신 (AnimStateMachine) 도입**:
    - 캐릭터의 대기(Idle), 걷기(Walk), 달리기(Run) 등의 모션 상태(State)와 상태 간의 전이 조건 규칙(Transition Rules)을 상태 그래프 구조로 정의할 수 있는 애니메이션 전이
      제어 아키텍처가 최초 설계 및 구현되었습니다.
    - 상태 머신이 캐릭터 조작 조건에 맞춰 실시간으로 판단하여 모션이 서서히 섞이는 크로스페이드 타이머와 각 애니메이션 레이어의 가중치(`blendWeight`)를 동적으로 조율하고, 최적의 믹싱 연산부로 흐름
      제어 값을 자동 전달해 줍니다.
* **가볍고 자연스러운 로컬 포즈 믹싱 (CPU-Side TRS Blending)**:
    - 3D 공간의 복잡한 뼈대 계층 구조를 연산하기 전 단계(로컬 공간)에서 캐릭터 모션들의 이동, 크기, 회전 값을 먼저 믹싱하므로 연산 속도가 가볍고 매우 정밀합니다.
    - 특히 회전 모션이 섞일 때 각도가 이상한 방향으로 꼬이지 않도록 **자동 최단 경로 구면 선형 보정(SLERP 최단 경로 보정)** 수식을 정확히 설계하여 적용했습니다.

## 4. 그림자 (Shadow) 시스템 변경

**"카메라 시야에 그림자 상자를 꽉 채워 해상도 낭비 없는 고화질 그림자를 만듭니다."**

야외 대형 씬이나 넓은 월드에서 카메라가 멀어지거나 움직일 때 그림자의 경계면이 뭉개지거나, 아예 나타나지 않는 현상을 해결하기 위한 파이프라인 개선이 적용되었습니다.

| 변경 및 개선 항목              | 기존 방식 (AS-IS)                               | 이번 버전에 새로 도입된 방식 (TO-BE)                                              |
|:------------------------|:--------------------------------------------|:----------------------------------------------------------------------|
| **정사투영 상자 범위 계산**       | 단순 카메라 거리에만 의존한 고정식 상자 크기 계산 (시야 바깥 해상도 낭비) | 카메라 뷰 프러스텀 8개 꼭짓점의 실시간 월드 공간 역투영 기반 맞춤 구속 (**Tight Fitting**)         |
| **그림자 강도(Strength) 제어** | 변경이 불가능한 획일적인 그림자 어두움                       | `DirectionalShadowManager.strength` 속성(기본값 `0.7`)을 통한 실시간 농도 제어       |
| **소프트 섀도우 필터 스케일**      | 튜닝이 제한적인 고정식 PCF 블러 반경                      | `DirectionalShadowManager.filterScale` 속성(기본값 `2.0`)을 통한 실시간 번짐 반경 조절 |

* **지터링 및 아크네 아티팩트 해결**:
    - 그림자가 닿는 면에 지저분한 가로줄이 생기는 섀도우 아크네(Acne) 현상과 물체 발밑에서 그림자가 붕 떠 보이는 현상을 해결하기 위해 최적의 그림자 바이어스(Bias) 값을 튜닝하여 반영했습니다.
* **CSM (Cascaded Shadow Maps) R&D 로드맵 수립**:
    - 시점 깊이에 따라 다단계 섀도우 해상도를 제공해 화질을 극한으로 끌어올릴 수 있는 CSM의 4단계 설계와 분석 연구가 완료되어 논의 폴더(
      `discussions/05_DirectionalShadow_Upgrade.md`)에 반영되었습니다.
