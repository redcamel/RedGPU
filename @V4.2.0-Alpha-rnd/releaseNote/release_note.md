# RedGPU V4.2.0-Alpha-rnd Release Note

이번 `V4.2.0-Alpha-rnd` 릴리즈는 대규모 성능 최적화(컬링), 그림자 품질/성능 튜닝, 캐릭터 입력 시스템 및 애니메이션 블렌딩 기법의 전면 고도화, 그리고 셰이더 내 전역 버퍼(Global
Buffer)의 체계적인 관리를 목표로 설계된 주요 업데이트입니다.

---

## 🚀 Key Highlights (주요 특징)

1. **애니메이션 및 컨트롤러 시스템 전면 고도화**
    - GLTF 기반 애니메이션 상태 머신(`AnimStateMachine`)과 뼈대 애니메이션 간 부드러운 전환을 지원하는 **모션 블렌딩(`gltfAnimationMotionBlending`)** 기법이 전면
      도입되었습니다.
    - 물리적 이동 및 조작을 손쉽게 연결할 수 있는 `SimpleCharacterController`가 추가되었습니다.
2. **프레임당 컬링 연산 최적화 (3대 컬링 기법 도입)**
    - 카메라 뒤쪽 오브젝트를 렌더 파이프라인 시작 전 걸러내는 **프러스텀 컬링 조기 탈락(Early-Out)** 기법을 도입했습니다.
    - 카메라 이동/회전 변화량을 감지하여 연산 부하를 프레임별로 분산하는 **인터리브드 컬링(Interleaved Culling)**을 추가했습니다.
    - 화면상 차지하는 픽셀 크기에 따라 자동 탈락시키는 **투영 면적 컬링(Screen Space Size Culling)**을 지원합니다.
3. **그림자(Shadow) 최적화 및 고품질 필터링 제어**
    - 불필요한 그림자 해상도 낭비를 방지하기 위한 **동적 프러스텀(Dynamic Frustum)** 연산 로직을 방향성 조명에 도입했습니다.
    - 런타임에 그림자 품질을 미세 조절할 수 있도록 **그림자 강도(Strength)** 및 **필터 스케일(Filter Scale)** 제어 속성을 정식으로 추가했습니다.
4. **글로벌 GPU 스토리지 버퍼 시스템 전면 리팩토링**
    - 글로벌 GPU 버퍼의 복사 및 효율적 갱신을 위해 `GlobalStorageBufferManager` 클래스를 신설하고 슬롯 관리 및 바인딩 경계 체크를 강화했습니다.

---

## 🛠️ Detailed Changes (세부 변경 사항)

### 1. Animation & Physics Controller (애니메이션 및 입력 컨트롤러)

* **SimpleCharacterController 추가**: 3D 캐릭터 메시에 직접 키보드/마우스 컨트롤러 및 카메라 오프셋을 매핑해 즉각적으로 3D 공간을 제어하는 헬퍼 클래스를 구현했습니다.
* **GltfAnimationMotionBlending & AnimStateMachine**:
    - 캐릭터의 걷기 ➡️ 뛰기, 대기 ➡️ 걷기 등의 모션 전환 시 선형 및 가중치 블렌딩을 적용하여 뚝뚝 끊기지 않고 물 흐르듯 변하도록 개선되었습니다.
    - 모션 상태에 따른 복잡한 애니메이션 흐름 제어를 위해 유한 상태 머신(FSM) 모듈을 내장했습니다.
* **키보드 입력 보강**: 브라우저 창에서 포커스가 해제(`blur` 이벤트)될 때 눌려 있던 모든 키 상태 버퍼를 안전하게 해제하여 입력 꼬임 버그를 해결하고 대소문자 키 동기화 성능을 개선했습니다.

### 2. Culling Optimizations (컬링 성능 극대화)

* **조기 탈락(Early-Out) 프러스텀 컬링**: 프러스텀 바운드 교차 검사 전, 카메라 후방 오브젝트를 사전 분류해 제거함으로써 연산 루프 시간을 대폭 단축했습니다.
* **인터리브드 컬링(Interleaved Culling) 도입**: 카메라의 움직임(이동 및 회전 속도)을 감지하여 컬링 체크가 필요 없는 정적 상태에서는 연산 빈도를 조절해 매 프레임 발생하는 불필요한 바운드
  체크 연산을 세이브합니다.
* **화면 크기 임계값 기반 컬링 (Screen Space Size Culling)**: `minScreenSpaceSize` 설정값보다 작게 렌더링되는 원거리의 아주 작은 오브젝트들을 조기 제외하여 렌더
  아키텍처의 병목 현상을 방지합니다.
* **그림자 프러스텀 예외 처리**: `castShadow` 속성이 참인 메시는 메인 프러스텀 영역 밖에 위치하더라도 그림자 투영 영역에 걸쳐 있을 가능성이 있으므로, 이를 고려하여 컬링 오판으로 그림자가 갑자기
  끊겨 보이는 오작동을 해결했습니다.

### 3. Shadow System Refactoring (그림자 렌더링 품질 제어)

* **동적 프러스텀(Dynamic Frustum) 그림자 맵**: 방향성 조명(DirectionalLight)의 뷰-프로젝션 행렬을 씬 경계에 맞게 동적으로 재구성하여 빈 공간에 그림자 맵이 낭비되는 문제를 막고
  그림자 선명도를 크게 개선했습니다.
* **그림자 상세 튜닝 속성 추가**: 조명별로 `shadowStrength` (그림자 어두움 농도) 및 `shadowFilterScale` (PCF 필터 부드러움 스케일)을 조절할 수 있도록 엔진 코어 및
  WGSL 쉐이더 코드를 수정했습니다.

### 4. Global Buffer & SSAO Restructuring (전역 버퍼 및 리소스 슬롯 통합)

* **GlobalStorageBufferManager 도입**: 기존의 `GlobalBufferManager`를 전면 개편하고, GPU 버퍼 복사 시 `CommandEncoderManager` 기반의 안전하고
  최적화된 복사를 수행하도록 재설계했습니다.
* **글로벌 SSAO 버텍스/프래그먼트 유니폼 버퍼 리팩토링**:
    - SSAO 처리를 위한 슬롯 바인딩과 쉐도우 깊이 리소스를 효율적으로 재배치하고 불필요한 바인딩 엔트리를 제거하여 바인드 그룹의 오버헤드를 낮췄습니다.
* **글로벌 프래그먼트 및 PBR 유니폼 버퍼 통합**:
    - 빌트인 재질(`isBuiltInMaterial`) 판정 로직과 글로벌 PBR 구조체를 쉐이더에 통합하여 머티리얼별로 중복 업로드되던 유니폼 구조를 단순화하고 효율을 증대시켰습니다.

### 5. Bug Fixes & Refactoring (버그 수정 및 리팩토링)

* **SkyBox NDC Z Far 클리핑 해결**: 초장거리의 하늘 영역이 GPU의 Far Clipping Plane에 의해 잘려 보이지 않는 오작동 방지를 위해 WGSL 쉐이더의 NDC 변환식을 개선했습니다.
* **ClusterLightManager 초기화 시점 해결**: 바인드 그룹 생성 과정에서 라이트 관리자(`ClusterLightManager`)의 내부 패스 변수들이 `undefined` 상태로 접근되어 앱이
  크래시되던 초기화 순서 버그를 생성자 주입식 패스 생성 구조로 전면 해결했습니다.
