# RedGPU Real-time Upscaling Roadmap (Resolution Innovation)

본 문서는 RedGPU의 렌더링 해상도를 지능적으로 복원하여 성능과 화질을 동시에 잡는 실시간 업스케일링 기술 로드맵을 정의합니다. RedGPU는 이미 업계 최고 수준의 TAA 인프라를 보유하고 있어, 이를 업스케일링으로 확장하는 데 최적화되어 있습니다.

---

## 🎯 전략적 우선순위 및 난이도 분석
저사양 기기에서도 하이엔드 그래픽을 구동하기 위한 핵심 최적화 기술의 개발 순위입니다.

| 우선순위 | 기능 명칭 | 임팩트 | 난이도 | 부하 정도 | 핵심 가치 |
| :--- | :--- | :---: | :---: | :---: | :--- |
| **1순위** | **TSR (Temporal Super Res)** | 🔴 최고 | 🟠 높음 | 🟡 보통 | TAA 인프라를 활용하여 저해상도 픽셀을 고해상도로 정밀 복원 |
| **2순위** | **Dynamic Res Scaling** | 🟠 높음 | 🟠 높음 | 🟢 낮음 | 부하에 따라 renderScale을 실시간 조절하여 프레임 안정화 |
| **3순위** | **Edge-aware Sharpening** | 🟡 보통 | 🟢 낮음 | 🔵 매우 낮음 | 업스케일링 시 발생하는 블러 현상을 엣지 감지 기반으로 복구 |
| **4순위** | **FSR 1.0 (Spatial)** | 🟢 낮음 | 🟡 보통 | 🟢 낮음 | 단일 프레임 정보만을 활용한 고성능 공간 업스케일링 |

## ✅ 현재 구현된 핵심 자산 (Existing Infrastructure)
RedGPU는 업스케일링을 위한 모든 '재료'를 이미 100% 갖추고 있습니다.

### 1. 렌더링 해상도 제어 시스템 (Render Scale)
- **현재 수준:** `RedGPUContext` 내에 `renderScale` 시스템 완비. (100%)
- **가치:** 저해상도 렌더링 → 고해상도 출력의 물리적 파이프라인 이미 구축됨.

### 2. 시간축 복원 데이터 (Temporal Data)
- **현재 수준:** `TAA.ts` 및 `Renderer.ts`에서 지터링 및 모션 벡터 완벽 관리. (95%)
- **가치:** 업스케일링의 핵심인 "픽셀 간 정보 추론 데이터"가 매 프레임 생성됨.

### 3. 고성능 샤프닝 필터 (Sharpening)
- **현재 수준:** `Sharpen`, `TAASharpen` (Contrast-aware) 구현 완료. (90%)
- **가치:** 업스케일링 후 발생하는 특유의 뭉개짐(Softness)을 즉각적으로 복구 가능.

## 🚀 기대 효과 시뮬레이션 (Expected Impact)
기술 완성 시 RedGPU가 얻게 될 정량적/정성적 이득입니다.

### 1. 성능 (Performance)
- **부하 감소:** 4K 출력 기준, 1080p 업스케일링 적용 시 셰이더 연산량 **약 60% ~ 70% 절감**.
- **프레임 향상:** Native 30fps 환경을 **80~90fps** 수준으로 즉시 격상 가능.

### 2. 품질 (Visual Quality)
- **재현율:** Native 해상도의 **약 90% ~ 95%** 수준 화질 도달.
- **안정성:** TAA 누적 덕분에 외곽선의 계단 현상이 거의 100% 제거되어 영화 같은 부드러움 제공.

## 🛠️ 기술적 구현 가이드 (Technical Analysis)
현재 프로젝트 구조상 체감 난이도는 **'보통~높음'** 수준입니다. (기초 공사가 이미 완료됨)

*   **남은 핵심 과제:**
    *   **Reconstruction Shader**: 1:1 매칭 TAA를 1:4 업샘플링 가중치 필터로 확장하는 전용 셰이더 구축.
    *   **UV Correction**: 저해상도와 고해상도 간의 픽셀 위치 오차를 0.001단위로 보정하는 정밀 로직.
    *   **Ghosting Control**: 해상도 확장 시 발생하는 이력 오차를 잡아내는 색상 클램핑 수치 튜닝.
