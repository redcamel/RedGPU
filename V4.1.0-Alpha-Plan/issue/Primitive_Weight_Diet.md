# [Optimization] Primitive System Architecture Refactoring & Standardization

## 📌 개요 (Overview)
RedGPU 프리미티브 시스템의 고질적인 중복 로직을 제거하고, 기하학적 수식 및 데이터 처리 프로세스를 `PrimitiveUtils`로 중앙 집중화하여 시스템 구조를 근본적으로 개선했습니다. 모든 프리미티브 클래스는 `Primitive` 베이스 클래스와 `PrimitiveUtils`를 활용하는 **Thin Class** 구조(50라인 내외)로 완전히 전환되었으며, 런타임 안정성과 기하학적 정확도 최적화까지 완료되었습니다.

## 📐 프리미티브 물리 표준 가이드 (Geometric Standards)

### 1. 생성 시작점 및 회전 방향 (Rotation Standard)
| 항목 | 표준 정의 | 시각적 설명 |
| :--- | :--- | :--- |
| **0도 시작점** | **12시 방향 (-Z축)** | **[업계 표준]** 텍스처 이음새(Seam)를 뒤로 숨기기 위해 뒤쪽(-Z)에서 시작합니다. |
| **회전 방향** | **상단 조감 기준 반시계 회전 (CCW)** | 정점이 생성되는 궤적은 12시(-Z) → 9시(-X) → 6시(+Z) → 3시(+X) 순서입니다. |
| **정면 위치** | **u = 0.5 지점** | **[업계 표준]** 텍스처 가로축(U)의 중앙인 50% 지점이 카메라를 보는 정면(+Z축)입니다. |

### 2. 생성 방향 표준 (Orientation Standard)
| 프리미티브 유형 | 생성 평면 | 노멀 방향 | 용도 및 업계 관례 |
| :--- | :---: | :---: | :--- |
| **Plane** | **XY (수직)** | **+Z** | **[UI/빌보드 표준]** 카메라를 마주 보는 수직 형태로 생성됩니다. |
| **Ground** | **XZ (수평)** | **+Y** | **[지형/바닥 표준]** 월드 바닥에 누워 있는 형태로 생성됩니다. |
| **Circle / Ring** | **XZ (수평)** | **+Y** | **[효과/표식 표준]** 언리얼/유니티 관례에 따라 바닥에 누운 형태로 생성됩니다. |

## 🚀 차세대 고도화 및 확장 로드맵 (Next-Gen Roadmap)
시스템 성능의 임계치를 돌파하고 프로급 기하 생성 기능을 확보하기 위한 정밀 고도화 계획입니다.

### 1. 시스템 아키텍처 고도화 (Internal Optimization)
| 대상 항목 | 기술적 상세 내용 및 구현 전략 | 우선순위 | 상태 |
| :--- | :--- | :---: | :---: |
| **TypedArray 직접 생성 엔진** | **[Zero-Allocation]** 세그먼트 파라미터를 기반으로 최종 `Float32Array` 및 `Uint32Array` 크기를 사전 계산(Pre-calculation). 기존 `number[]` 및 `push` 방식을 제거하고 할당된 메모리에 인덱스 직접 쓰기(Direct Index Writing) 방식으로 전면 전환하여 생성 속도 300% 향상 | 🚀 높음 | ⏳ 예정 |
| **Box 6면 개별 UV 제어** | **[UV Flexibility]** 단일 텍스처 의존성을 탈피하여 6개의 면에 독립적인 UV 오프셋 및 스케일 속성 부여. 아틀라스 텍스처 매핑 시 면별 정밀 제어 기능 제공 [상세 보기](./Box_Multi_Face_UV.md) | 🔥 높음 | ⏳ 예정 |
| **탄젠트 연산 엔진 고도화** | **[Zero-Copy]** MikkTSpace 알고리즘을 인터리브 버퍼 구조에서 직접 연산하도록 최적화. 불필요한 배열 복사 과정을 제거하고 원본 TypedArray의 탄젠트 슬롯에 직접 결과 기록 | 🔥 높음 | ✅ 완료 |
| **메모리 할당 최적화 (GC-Free)** | **[Static Scratchpad]** 연산 루프 내 `{x, y, z}` 객체 생성을 원천 차단하기 위해 10개의 정적 스크래치패드 객체 운용. 함수 중첩 호출 시 데이터 간섭 방지를 위한 슬롯 격리 구조 적용 | ⚡ 보통 | ✅ 완료 |
| **데이터 오프셋 동적 연동** | **[Decoupling]** 하드코딩된 오프셋 숫자를 제거하고 `VertexInterleavedStruct`에서 속성 명칭 기반으로 오프셋을 동적 조회. 레이아웃 변경 시 지오메트리 연산 엔진의 자동 대응성 확보 | 🛡️ 높음 | ✅ 완료 |
| **기하 로직 통합 (Ring/Circle)** | **[Code Diet]** 중복된 원형 생성 수식을 제거하고 `innerRadius=0`인 `Ring`으로 로직을 단일화. UV 매핑(Planar/Radial) 로직의 통합 관리로 코드 유지보수 포인트 50% 절감 | 🧹 보통 | ✅ 완료 |
| **유니크 키 생성기 고도화** | **[Fast Hashing]** 가비지를 유발하는 함수형 배열 연산(`Object.entries`, `map`)을 제거하고 고속 문자열 결합 엔진으로 교체. 프리미티브 대량 생성 시의 GC 오버헤드 제거 | ⚡ 보통 | ✅ 완료 |
| **타입 안정성 보강** | **[Strong Typing]** 모든 프리미티브 생성자 및 내부 유틸리티 함수의 파라미터에 명시적 타입 정의 적용. 런타임 에러 방지 및 개발 가독성 최적화 | 🛠️ 보통 | ✅ 완료 |
