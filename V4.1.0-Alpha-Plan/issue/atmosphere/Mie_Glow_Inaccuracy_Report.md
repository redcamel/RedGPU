# [Report] Mie Glow 시각적 아티팩트 및 물리적 구현 이슈 분석

## 1. 개요
현재 `SkyAtmosphere` 시스템의 하이브리드 Mie Glow 구현에서 글로우가 지나치게 강하게 나타나고, 특정 각도에서 범위가 단절되는 시각적 아티팩트가 보고되었습니다. 셰이더 코드 분석 결과, 위상 함수(Phase Function)의 차감 로직과 에너지 보존 수식에서 수학적 오류를 확인했습니다.

---

## 2. 핵심 문제점 분석

### 2.1 위상 함수 차감으로 인한 범위 단절 (Phase Cutoff) - (**해결 완료**)
*   **조치**: `diffPhase` 차감 로직을 제거하고, `sharpPhase`를 직접 합산하도록 수정하여 컷오프 현상을 근본적으로 해결했습니다.

### 2.2 에너지 보존 수식의 불일치 (Energy Inconsistency) - (**해결 완료**)
*   **조치**: LUT의 누락된 에너지 비율(`mieGlow`)과 분석적 글로우의 위상 함수를 물리적으로 일치시켜, 전체 에너지가 정규화되도록 교정했습니다.

### 2.3 태양 투과율 근사의 정확도 부족 (Transmittance Approximation)
*   **코드**: `getTransmittance(..., mappingH, params.sunDirection.y, ...)`
*   **원인**: 글로우 계산 시 카메라 위치에서의 수직 투과율만 참조합니다.
*   **문제**: 실제 대기 투과율은 태양의 고도와 시선 방향의 경로 길이에 따라 복합적으로 변해야 합니다. 이 단순화된 모델은 태양이 지평선에 걸릴 때(노을) 글로우가 물리적 수치보다 너무 밝게 유지되게 만듭니다.

### 2.4 과도한 피크 휘도 (High Peak Intensity)
*   **원인**: `mieHalo`(0.9) 수치가 매우 높아 태양 중심점의 휘도가 기하급수적으로 증폭됩니다. 여기에 `sunIntensity`가 중복 적용되면서 픽셀 값이 `rgba16float` 범위를 쉽게 초과하여 시각적으로 "타버리는" 현상이 발생합니다.

---

## 3. 개선 권장 사항 (Proposed Fixes)

1.  **위상 함수 합성 로직 교정**: 
    *   `diffPhase` 차감 로직을 제거합니다.
    *   분석적 글로우 계산 시 `sharpPhase`를 직접 사용하고, LUT에서 누락된 Mie 에너지(`mieGlow`)를 보충하는 정교한 가중치 모델로 변경합니다.
2.  **Dual Phase 정규화 적용**:
    *   `phaseMieDual` 구조를 합성 단계에서도 엄격히 적용하여, 모든 각도에서 총 에너지가 1이 되도록 정규화합니다.
3.  **투과율 모델 정밀화**:
    *   단순 카메라 위치 투과율 대신, 수평선 부근의 감쇄를 반영하는 향상된 근사식을 사용합니다.
4.  **피크 억제 (Soft-Knee)**:
    *   중심부의 과도한 밝기를 물리적으로 타당한 수준에서 부드럽게 억제하는 클램핑을 도입합니다.
