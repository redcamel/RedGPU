# [Issue] 태양광 강도 정규화 및 IBL 시각적 일관성 확보

## 1. 개요
대기 산란 시스템에서 태양광이 비정상적으로 강하게 표현되어 화면이 타버리는(Overexposure) 현상과, 배경 하늘과 오브젝트(Aerial Perspective) 간의 태양 Glow 표현이 일치하지 않는 문제를 해결했습니다.

## 2. 상세 문제점 및 원인 분석

### 2.1 다중 산란 중복 배율 (Multi-Scattering Double Scaling)
*   **원인**: `skyViewScatMult` 파라미터가 산란 계수 계산 시 이미 반영되어 있음에도 불구하고, 다중 산란광(`msScat`)을 합산하는 과정에서 계수가 중복으로 곱해지고 있었습니다.
*   **현상**: 산란 강도를 높일수록 대기광이 기하급수적으로 밝아져 물리적인 에너지 보존 법칙이 깨지는 현상이 발생했습니다.

### 2.2 조도(Irradiance) 중복 합산
*   **원인**: 반사 큐브맵(`reflectionTexture`)에는 이미 태양 본체와 Mie Glow가 구워져 있습니다. `AtmosphereIrradianceGenerator`가 이를 적분하여 조도를 만들 때, 셰이더 마지막 단계에서 분석적 모델로 태양광을 한 번 더 더해주고 있었습니다.
*   **현상**: IBL을 통한 디퓨즈 라이팅이 실제보다 2배 이상 밝아져 씬의 오브젝트들이 과하게 밝게 렌더링되었습니다.

### 2.3 포스트 이펙트(AP)의 Glow 누락
*   **현상**: 배경 하늘에는 날카로운 Mie Glow가 있지만, `computeCode.wgsl` (Aerial Perspective)에는 해당 로직이 누락되어 있었습니다. 
*   **결과**: 태양 앞에 있는 물체들은 Glow 효과를 받지 못해 배경 태양만 붕 떠 보이는 시각적 이질감이 발생했습니다.

## 3. 해결된 아키텍처 (Implemented Solution)

### 3.1 다중 산란 로직 정규화
*   `skyAtmosphereFn.wgsl`의 `integrateScatSegment`에서 `msScat` 계산 시 불필요한 중복 배율을 제거했습니다. 
*   이제 산란 배율은 선형적으로 동작하며 물리적인 투과율 감쇄와 정확히 연동됩니다.

### 3.2 IBL 에너지 밸런스 최적화
*   **조도 중복 제거**: `atmosphereIrradianceShaderCode.wgsl`에서 분석적 태양광 합산 로직을 제거했습니다. 이제 큐브맵 적분값만 사용하여 중복 없는 정확한 간접광 강도를 제공합니다.
*   **단위 강도 기반 Glow**: `getMieGlowAmountUnit` 함수를 도입하여 모든 셰이더가 동일한 물리 공식을 공유하도록 통합했습니다.

### 3.3 시각적 일관성 확보 (Consistency)
*   **AP Glow 추가**: `computeCode.wgsl`에 Mie Glow 계산을 추가하여, 물체와 카메라 사이의 대기에서도 태양 산란광이 올바르게 표현되도록 개선했습니다.
*   **파라미터 캘리브레이션**: `solarIntensityMult` 기본값을 `20.0`으로, `mieHalo`를 `0.9`로 조정하여 톤 매핑 환경에서 가장 사실적인 결과가 나오도록 기본 밸런스를 맞췄습니다.

## 4. 기대 효과
*   씬 전체의 조명 밝기가 물리적으로 예측 가능한 범위 내로 안정화됨.
*   배경 하늘과 물체 사이의 대기 표현이 완벽하게 일치하여 공간감이 향상됨.
*   과도한 에너지 집중으로 인한 수치적 불안정성(Fireflies) 및 밴딩 현상 억제.
