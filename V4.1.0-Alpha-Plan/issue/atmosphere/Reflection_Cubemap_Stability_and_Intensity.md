# [Issue] 반사 큐브맵(Reflection Cubemap) 물리적 정합성 및 수치 안정성 개선

## 1. 개요
IBL(Image Based Lighting)의 소스인 반사 큐브맵이 실제 씬 대비 비정상적으로 밝게 생성되던 문제를 해결하고, 태양 주변의 수치적 노이즈(Fireflies)를 제거하여 안정적인 반사 데이터를 확보했습니다.

## 2. 상세 문제점 및 분석

### 2.1 강도 중첩 문제 (Double Intensity)
*   **원인**: 반사 큐브맵 자체에 `sunIntensity`를 곱해버릴 경우, 이를 사용하는 PBR 머티리얼이 태양광을 직접광(Direct Light)으로 별도 계산할 때 에너지가 이중으로 합산되는 현상이 발생했습니다.
*   **조도 생성기 충돌**: `AtmosphereIrradianceGenerator` 역시 태양 기여도를 수학적으로 별도 합산하므로, 텍스처에 구워진 태양광과 중복되어 씬 전체가 과노출되었습니다.

### 2.2 수치적 불안정성 (Fireflies)
*   256x256 해상도에서 극단적으로 높은 강도(수만 단위)의 픽셀이 1픽셀 미만 크기로 존재할 때, 샘플링 오차에 의해 화면이 지저분하게 튀는 현상이 발생했습니다.

## 3. 해결된 아키텍처 (Final Implementation)

### 3.1 단위 강도 기반 설계 (Unit Scale Restoration)
*   **변경 내용**: 반사 큐브맵 생성 시 모든 `sunIntensity` 배율을 제거했습니다.
*   **결과**: 반사 큐브맵은 이제 **물리적인 색상 비중(Ratio)만 담고 있는 단위 강도 상태**를 유지합니다. 최종 밝기는 큐브맵을 사용하는 시점(PBR Shader 등)에서 조절하도록 역할을 분리하여 중복 계산을 원천 차단했습니다.

### 3.2 수치 안정화 및 노이즈 억제
*   **Mie Glow 안정화**: 비등방성 `g` 값을 **0.80**으로 유지하여 날카로운 산란광 느낌을 살리되, 점 현상이 발생하지 않도록 밸런스를 맞췄습니다.
*   **태양 본체 비율 조정**: 태양 본체 강도를 `solarIntensityMult * 0.1`로 조정하여, 하늘색 대비 태양의 상대적 비중을 물리적으로 정합하게 맞췄습니다.
*   **Soft Clamping**: 수치적 튐 방지를 위해 단위 강도 기준 `100.0`으로 클램핑 범위를 대폭 축소하여 HDR 안정성을 확보했습니다.

### 3.3 지평선 및 지면 차폐 강화
*   `horizonElevation`을 물리적으로 계산하여, 지면 방향으로 태양 빛이나 산란광이 누수(Leak)되지 않도록 엄격하게 마스킹했습니다.

## 4. 기대 효과
*   금속 재질의 물체에서 태양과 대기 반사가 타버리지 않고 선명하고 깨끗하게 표현됨.
*   조도 큐브맵(Irradiance) 생성 시 태양 기여도가 중복되지 않아 씬 전체의 조명 밸런스가 안정화됨.
*   낮은 큐브맵 해상도에서도 파이어플라이 현상 없이 부드러운 HDR 반사 데이터 제공.
