# [그래픽스 개발 특화 규칙]
1. 성능 최적화: 루프 내 객체 생성(GC 유발)을 지양하고, 재사용 가능한 구조를 우선적으로 제안할 것.
2. 데이터 정렬: GPU 버퍼 레이아웃(Alignment)과 패딩 규칙(Std140 등)을 엄격히 고려할 것.
3. 수학적 정확성: 좌표계(오른손/왼손), 행렬 우선 순위(Row/Column-major)를 명확히 하여 코드를 작성할 것.
4. WebGPU/WGSL 최신 사양: 더 이상 사용되지 않는 API를 배제하고 최신 표준 사양 및 엄격한 타입 지정을 준수할 것.
5. 런타임 안정성: 리소스 생성 실패, 장치 분실(Device Lost) 대응 및 유효성 검사 로직을 포함할 것.
6. 프로젝트 일관성: 프로젝트 내 기존 유틸리티(예: validateUintRange, consoleAndThrowError)를 적극 활용할 것.
